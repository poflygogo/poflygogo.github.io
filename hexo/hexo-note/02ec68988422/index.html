透過 Windows 在 Github 上架設 Hexo 的流程

發表於2025-08-03|更新於2025-08-03
|瀏覽量:

前言

架設一個 Hexo 網站本身不需要會寫任何程式碼，都用自然語言就可以了。

以我個人來說，我只會後端語言，例如 python, java, kotlin……之類的，對於製作網站那是一竅不通，爬蟲確實是會寫，分析 html 我確實知道怎麼做，解析封包也還行，但真要讓我自己搓一個網站，那是不可能的，完全不可能。

Hexo 幫忙解決了這問題，讓我能夠在不寫任何 Code 的情況下寫文章，全程基本上不寫任何程式碼是真的，但如果真的讓一個完全不會寫 Code 的人處理這套過程還是有一點挑戰的，過程中得要能耐著性子閱讀各種文檔，官方的、非官方的都要來一點。

我在這方面稱不上專家，但分享一點成功經驗還是可以的。

流程

可以用條列式的方式簡單描述成這樣

  1. 配置環境，Hexo 本身只是一個框架，而 Hexo 需要 Node.js 和 git，這些都是需要先準備好的。

  2. 準備 github 帳號，要架設在 github 上，得先有帳號，有開啟對應的倉庫。

  3. 發布網站

配置環境

安裝 Node.js

直接去官網下載即可

連結: Node.js — 下載

須注意的是官網用較大的版面建議透過 docker 安裝，這是對的，也是更推薦的做法，目的是不汙染環境，但這篇文主要是給不會寫程式的人參考，這時候 docker 除了增加複雜度外幫助不大。(若完全看不懂這段在說什麼，代表你不需要 docker，可以直接看下一句)

我們直接下載底下提供的 msi 檔案，執行後不斷確認即可，它會檢查你電腦缺少的元件並自動安裝， 需要一點時間。

如果你有在寫程式，不推薦使用 msi，應該直接用 docker，因為這 msi 檢查環境的方式非常粗糙，容易破壞你設備上的環境，到時候要處理反而讓自己更加困擾。

筆者偷懶使用了它，這 msi 執行檔直接把我原本的 python 環境給取代掉了，之後處理花了一點時間，坦白說我是很後悔的，如果有一點底子的就去用 docker 吧。

操作完後我們需要檢查 Node.js 是否已安裝成功，Node.js 本身並沒有一個專門的圖形化介面，所以你直接執行它的話(若你能找到執行檔)，只會看到你的 PowerShell/CMD 被打開了。不用擔心，這邊你只要複製貼上即可。

首先，要先開啟你電腦上的 CMD 或 PowerShell (你有 bash 想用 bash 也可以)，按下 win + R 這個組合鍵，然後輸入 cmd 後確認，你會看到像這樣的畫面:

node 就是 node.js 的本體，在 cmd 中我們會透過輸入 node 呼叫它，但你不需要在意這個，因為我們只需要見到他這一次而已，之後都會透過 npm 操作。

npm 則是 node.js 的套件管理工具，可以用來快捷安裝一些東西，待會我們會需要用他做很多事情，所以如果你的 npm 啟動失敗一定要處理。

一直按下一步就可以。

安裝結束後一樣要檢查是否安裝成功，鍵盤按下 win+R ，輸入 cmd 後確認，在 cmd 中輸入下面這條指令:

有顯示一長串像這樣的文字那就是安裝成功了。

git 是檔案管理工具，主要用來追蹤檔案的變化狀態，如果你有寫程式的基礎，但沒使用過它，可以趁機會好好學一學，特別是 ai 時代，git 的重要性會越來越高。

倘若你根本不寫程式，那你就理解成 Hexo 需要的前置工具就好，不需要真的會用，當作不存在即可，盡管它確實可以幫助你管理網站的狀態，但你應該不會想用它，因為它學起來是需要不少時間實踐的。

安裝 Hexo

執行這步時，請確保前面的事情你都有做了

這次我們要在 cmd 中輸入比較複雜一點的指令

如果你不放心，我把這條指令做的事情告訴你

  • npm 意思是呼叫 npm 這個工具，說明我們要使用它

  • install 是 npm 用來安裝套件的指令，告訴 npm 我們想要安裝某個東西。

  • hexo-cli 就是本文的主角 Hexo，我們得透過這個方式安裝並使用它。

  • -g 意思是把這個套件放到全域環境中，確保無論你 cmd 在那個資料夾層級都可以使用它，如果你本身有寫程式的背景，很清楚你在做什麼，可以選擇不加。這邊為了避免困擾，建議是放進去。

整合成一句話就是「使用 npm 這個工具安裝 Hexo」

在此恭喜你已經成功把 Hexo 安裝到你的電腦上。

建立專案

這時可以著手架設你的網站了，想讓網站動起來，你得先有東西，Hexo 只是輔助你架設網站，並不會憑空產生內容，內容需要你自己準備。

首先，選擇好一個風水好的位置建立資料夾，你以後處理網站相關的東西就只會在這操作了，所以風水請挑好。

有兩個做法:

  1. 直接使用下面這條指令，會在你指定的路徑建立一些必要的檔案，注意路徑務必要輸入正確。

    例如: hexo init C:\Users\user\Documents\Blog

  2. 另一個做法是讓 cmd 移動到目標資料夾，然後輸入下面的指令

    要讓 cmd 移動到目標資料夾的方式有很多，這邊提兩個:

    • 可以在指定資料夾中按下右鍵開啟 cmd

    • 也可以透過下面這個指令前往指定資料夾

一樣，我會解釋這條指令在做什麼，讓你放心使用。

  • hexo: 意思是我們要呼叫 Hexo 這個工具作一些事情

  • init: 意思是我們要初始化專案，執行後會在專案內建立一些必要的檔案，他們都是用來幫助你架設網站的東西。

如果能做到這一步，我要恭喜你，對你來說可能最難的部分已經完成，你可以嘗試執行下面這條指令，然後開啟瀏覽器，使用這個網址 http://localhost:4000 預覽你的網站

這條指令可以讓你在本地端(不用網路的環境)查看網站現在的樣子，裡面應該有一篇名為 Hello World 的文章，可以當範本用，裡面有一些常用指令，有興趣的話可以參考。

你可以現在就著手撰寫文章，但我更傾向讓人先看到成果，這樣才不會太傷自信。

但目前你的網站還沒辦法讓大家看到，因為你還沒有架到網路上，所以我們要進入下一個步驟。

前往 GitHub 並建立對應的倉庫

可以透過這個連結前往 GitHub: https://github.com

說明一下這步驟的目的，我們現在要使用的是一項名為 GitHub Page 的服務，是 GitHub 免費提供給大家使用的，用來架設靜態網站的服務。

一般來說，如果我們要架設一個大家都能瀏覽的網站，需要一台主機長時間工作，或是申請線上伺服器，這些通常都需要一點錢才有辦法處理。另一方面，還需要購買域名，這樣別人才有辦法透過輸入網址找到你的網站……需要處理的事情還有很多。

但 GitHub Page 幫我們處理好這些事情了，它提供一個你專用的網址，幫你處理好網站安全，甚至幫你把主機維護費出了，你只需要專注於你的內容即可，這就是我們選擇它的原因。

帳號申請的部分我就不細說怎麼操作了，看不懂英文的話，教學很多，整體流程和一般平台申請帳號的方式差不多，就不贅述。

GitHub 的倉庫名為 Repository，我們會透過一個個的倉庫管理每個不同的專案，包含你想架設的網站。

第一步當然是建立一個新的倉庫了，登入後點擊網站左側選單的 New

在這裡設定倉庫名稱，名字不能亂打，有固定格式，這樣才有辦法被 github 識別為你想架設的網站，格式為 <你的帳號名稱>.github.io，帳號名稱請自行修改，例如這樣 sample.github.io

檢查後確認無誤，直接按確認即可，其他設定不需要調整。

如果你有使用 Github 的經驗可能會想調整一些東西，這邊建議你注意確保專案是公開的，其他部分有需要的話可以參考 GitHub Page 的說明文檔。

把網站部屬到 GitHub 上

有三個步驟:

第一步: 修改配置文檔

現在回到你的電腦上，打開你的專案資料夾根目錄，裡面應該會有一個名為 _config.yml的檔案，使用任意文字編輯器打開它，例如記事本就可以，或者你有習慣的工具就用你習慣的。

打開後會看到一大堆文字，都英文，不用擔心，不要求你真的全部都要看懂，我們需要你找到 deploy 下的這段文字:

複製貼上過去就可以，repo 那個欄位請，請設定成我們剛剛設定的那個 github 倉庫網址，但後面記得要添加 .git，像這樣: sample.github.io.git

第二步: 安裝 Git 相關套件

回到專案的根目錄，打開 cmd，輸入下面這條指令安裝必要的套件

  • npm 是用來管理 Node.js 套件的工具，我們已經見到它很多次

  • install 意思是我們要讓 npm 安裝某個套件

  • hexo-deployer-git 是我們這次要安裝的東西，作用是讓 Hexo 能夠協助我們透過 git 部屬網站。

  • --save 會把套件的資料存起來，放在一個名為 package.json 的檔案中，這可以協助你未來如果想在不同設備操作 Hexo 時，簡化流程，不用手動操作一堆東西。

第三步: 部屬你的網站

現在輸入下面這條指令，就會把你的專案上傳到 github 上

現在可以回到 github 查看你的倉庫，確認是否成功，如果沒有任何檔案出現，代表可能網址寫錯了，請檢查 _config.yml 找到 deploy 的部分，看看網址有沒有寫錯。

你可以透過這個網址查看你剛剛架上去的網站 https://<你的帳號名稱>.github.io 再次確認是否有發布成功。

這樣就算成功部屬網站到 GitHub 了，恭喜你。
